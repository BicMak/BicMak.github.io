## 1. Page fault
- 프로세스가 메모리 페이지를 사용하려는데 준비되지 않았으면 페이지 폴트가 발생
- 발생하면 커널이 페이지를 준비하기 위해 프로세스로부터 CPU 제어권을 획득
### Minor 페이지 폴트
- Minor 페이지 폴트는 원하는 페이지가 실제로는 메인 메모리에 있지만 MMU가 그 위치를 모를 때 발생
- 커널이 MMU에게 페이지 정보를 알려주고 프로세스가 계속 진행됨
> ex) 최초로 가상주소를 통해서 MMU에 접근할때 발생 

### Major 페이지 폴트
- Major 페이지 폴트는 원하는 메모리 페이지가 메인 메모리에 전혀 없을 때 발생
- 커널이 디스크나 다른 느린 저장 메커니즘에서 로드해야 한다는 의미
- 로드 과정자체가 비효율적이라서 성능하락의 주범
- 메모리가 부족해지면, 커널이 새 페이지를 위한 공간을 만들기 위해 작업 메모리 페이지를 디스크로 스왑 아웃하기 시작하고 이는 스래싱(thrashing) 문제가 됨

### Segmetation fault/bus error
- 프로그램이 접근하면 안 되는 메모리 부분에 접근하려고 했고, 운영체제가 그것을 종료
- bus error는 프로그램이 어떤 메모리에 해서는 안 되는 방식으로 접근하려 했다는 의미
- 아래 2가지 경우를 가지고 에러를 판단

1. **"이 가상 주소가 애초에 유효한 주소인가?"** (VMA 체크)
    - 이 프로세스의 코드/데이터/힙/스택/mmap 영역 중 어디에 속하는가?
    - 없으면 → 즉시 SIGSEGV
2. **"접근 권한이 맞는가?"** (권한 체크)
    - 읽기 전용인데 쓰기? → SIGSEGV
    - 실행 불가인데 실행? → SIGSEGV
