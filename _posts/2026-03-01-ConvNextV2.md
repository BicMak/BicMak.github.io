## 2. Model Detail

### 2.1 Fully Convolutional Masked Autoencoder

- The process involves randomly **removing 60% of $32 \times 32$ patches** from the original input image. 
- Masking is performed at the granularity of $32 \times 32$ pixel patches. For a standard $224 \times 224$ input, there are approximately 49 patches; 60% of these are masked, and this masking is applied to both the **residual skip connection** and the **convolutional operations**. 
- This design was implemented to facilitate **patch-level attention** within the convolutional framework. 
### 2.2 Global Response Normalization (GRN)

-  ConvNets often suffer from a chronic issue known as **feature collapse**, where feature maps become saturated or "dead," leading to redundant activations across different channels. 
- Given an activation tensor $X \in \mathbb{R}^{H \times W \times C}$, let $X_i$ be the $i$-th channel's feature map reshaped into an $HW$-dimensional vector. The $L_2$ norm is then calculated across these channels. 
- Based on the principle that **higher norm values represent more diverse features**, this method applies weights to dynamically emphasize specific layers and contrast channels against one another. 

```python
# gamma, beta: learnable affine transform parameters # X: input of shape (N,H,W,C)

gx = torch.norm(X, p=2, dim=(1,2), keepdim=True) 
nx = gx / (gx.mean(dim=-1, keepdim=True)+1e-6) 
return gamma * (X * nx) + beta + X
```

- A key advantage is that it is **input-dependent**, meaning it reflects the input image dynamically and applies consistently across different images.

### 2.3 ConvNeXT-V1(Modernization Strategy)

- This involves a CNN block created by adopting modern architectural techniques from **Swin-Transformers** into the traditional ResNet structure.

1. **Block Ratio Adjustment:** The block distribution across stages was changed from the traditional ResNet-50 ratio of **(3, 4, 6, 3)** to match the Swin-Transformer's ratio of **(3, 3, 9, 3)**.

2. **Inverted Bottleneck Structure** :![alt text](/assets/img/2026-03-01-ConvNextV2/image.png) Borrowing from the Transformer's block sequence (Linear → Self-attention → Softmax), the compute ratio was inverted. Since most Transformer computation occurs in the attention mechanism, the resource allocation for **$1 \times 1$ kernel channel-wise operations** was significantly increased.
	
3. **Large Kernels:**![alt text](/assets/img/2026-03-01-ConvNextV2/image-1.png) While standard $3 \times 3$ kernels excel at local feature extraction, they struggle to capture **long-range correlations** like Transformers do. Simply stacking more layers can lead to "feature dilution" where values saturate. To solve this, the model uses **depthwise convolutions** with larger kernels to capture patch-wise correlations, effectively applying Transformer-style attention concepts to a CNN.
